Chapter 3

The Building Block of ANN

Early deep learning researchers in the 1970s took inspiration from a neuron, the building block of our brain, to create an algorithm called a perceptron.

Therefore, to understand what a perceptron is, let’s first take a look at a neuron.

A neuron is made up of 3 main parts:

  • Dendrites: Spread out receivers of the neuron that takes in some stimulus from other neurons.

  • Soma: The “core” of the cell where the nucleus is situated at, and it decides if a signal will be sent to other neurons.

  • Axon: Long extending neck of the neuron that passes a signal to all connected neurons.

From Neuron to Perceptron

The perceptron, similarly, also has 3 major parts:

  • Inputs [Dendrites]: inputs take in values from an arbitrary number of sources and pass it to the perceptron.

  • Perceptron [Soma]: the perceptron algorithm takes in the inputs and assigns a weight to each input. It then calculates the weighted sum of all inputs (the sum of all weight * input), then adds a bias on to of that.

  • Output [Axon]t: the output of the perceptron spits out the result of the calculation.

Between [two neurons], the biological axon-to-dendrite connection is simplified to an edge, and signals transmitted are represented with numbers.

The input on top is connected to the central neuron with a thick edge, thus the signal is amplified by a factor of 2 when it reaches the neuron.

The input at the bottom, however, has a thinner connection to the neuron, thus the signal is reduced to 2, resulting in a final signal of 2 + 2 = 4 received, instead of 3 + 2 = 5.

The thickness of the edge is called weight in machine learning.

Next, the neuron processes the signal by shaking itself. In this case, doing so incremented the signal by 1.

The “shakiness” of a neuron is called bias, which can both increase or decrease the value of the signal by any set amount.

Finally, an activation function is used to simulate how "fired" the neuron is.

ReLU, a type of activation functions, is used here. It preserves the signal value if it is 0 or greater.

In our case, a positive signal would fully fire the neuron, so the value 5 is preserved and becomes the final output of the neuron.

Activation Functions

Early researchers use these perceptrons to model how human beings think. In spite of early successes, they quickly ran into a problem. When a researcher wants to predict how likely an animal with 2 legs and 2 arms is a human, the probability the model gives back would often be above 100% or below 0%.

Intuitively, this isn’t possible, but to the machine, it is just trying to estimate a number, and it does not know if that number is meaningful.

To tackle such problems, the researchers put an activation function behind the output of the perceptron, which can control the output range of that perceptron.

For example, ReLu and Sigmoid are functions that are commonly used. These functions allow the perceptron to limit its output to a specific range so that it would not spit out invalid outputs.

ReLu, which we have already seen, makes the final output value non-negative. It preserves the value of the output as long as it is greater than 0. If the output is equal to or below 0, it would simply output 0. This function is by far the most popular activation function, and you can find it being used in areas like Computer Vision and Speech Recognition.

Sigmoid calculates the probability of two things, for example, if you want to classify if an image is a cat or isn’t a cat (this is called binary classification), you would want to use a sigmoid activation function, which transforms the output value into a probability (a number between 0 and 1).
