Chapter 6

Feed-forward Neural Networks in Action

Wondering how the module in chapter 0 actually works? Harnessing the knowledge you have acquired so far, let's take a look at how you can build such network to recognize handwritten digits!

Scroll

In this chapter, we will explain how to make a practical model that will recognize hand-written digits, just like the one in Chapter 0. We will be using the MNIST dataset, a dataset of hand-written images compiled by researcher Yann LeCun. We will be splitting 80% off the dataset to use for training and the other 20% for checking our accuracy.

In the input, we flattened our 28 x 28 MNIST image to an array of 784 pixels.

Each pixel has a value from 0 to 1 indicating how black that pixel is. 0 is white, 1 is black, and anywhere between them is a shade of gray.

The first layer takes in the input of size 784 and transforms it into an array of size 500, so we have 500 numbers that will represent a transformation of the original image. For a multi-layered network to work well, we need to break linearity, thus we use ReLU as an activation function to achieve that.

Since we are working with classification, we would need to use cross-entropy loss. This would require us to input probabilities for each class, rather than an arbitrary value. To do this, we use a softmax activation function in the end. Softmax is a hyped up version of sigmoid wherein it generates not only probability of one thing, but multiple things. This way, we will get the probability of each digit we can use to calculate loss.

Then we take the array of 500 values and transform that into an array of 10 values, each corresponding to digits “0” to “9”.

The value that is highest, signaling the one that the model thinks is most probable, will be our prediction.

We then split them by batches of 100 and feed them into the model for training. During training, the model uses existing parameters to generate a tentative answer (this is usually called forward propagation).

Then, after seeing the answer and computing the loss (how different am I from the real answer), it goes back into the network and updates the parameters to get even closer to the real answer (this is usually called backpropagation).

To train the model, we would need to first initialize the values inside the model. In this particular case, we will initialize them to something random clustered around 0, which is usually a good starting point.

To train it, we are using SGD (stochastic gradient descent) as introduced before, with a learning rate of 0.7. This learning rate is chosen by experiment, as it seems to work best for our model. Usually, you should try out a few different values (depending on the data and the loss function, the learning rate is usually under 1, and bigger than 0.0001) before settling on a reasonable one.

After going over the dataset 2 to 3 times (epochs), our model is good enough to be used. In our model, we would be able to achieve around 97% accuracy with only 2 runs over the dataset, and on an average laptop, it would run in under a minute (excluding the time downloading the dataset).
